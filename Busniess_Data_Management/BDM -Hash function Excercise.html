BDM - Hash Function Exercise¶

Hash Function :¶

A hash function is any function that can be used to map data of arbitrary size to data of a fixed size. The values returned by a hash function are called hash values, hash codes, digests, or simply hashes.¶

Libraries Used -¶

1-> Hashlib - Secure hashes and message digests :¶

This module implements a common interface to many different secure hash and message digest algorithms.
Included are the FIPS secure hash algorithms SHA1, SHA224, SHA256, SHA384, and SHA512 (defined in FIPS 180-2) as well as RSA’s MD5 algorithm (defined in Internet RFC 1321).

2 -> Collections ( Counter) -¶

This module implements specialized container datatypes providing alternatives to Python’s general purpose built-in containers, dict, list, set, and tuple.¶

Components:¶

1) Lists :¶

sha_hex -> For storing the hexcadecimal code output of sha2 function. final_sha -> For Storing the remainder ater the Moduls with N names -> for storing the names.

collusions -> to store the count of each element.

2) Function :¶

encrypt_string() : for returning the sha2 64bit output string. it will input the names and give 64bit string as output

hexdigest() -> to get the generated sha2 String.

encode() -> For encoding the names string.

3) Variables:¶

Hex1 -> gets the output of encrypt_string Function. sha_signature -> for storing the sha2 output string

WorkFlow¶

1) we have selected random 10 names
2) Get these names Encrypted and store in sha_hex list
3)Parse the Strings to Integers
4) Take Mod of N with each generated string in sha_hex and store it in final_sha
5) Get the count of each Element of the final_sha using Counter
6) Print collusions.
